# 📘 คู่มือการใช้งานระบบจองห้องประชุม (Room Reservation System)

ยินดีต้อนรับสู่ระบบจองห้องประชุมเวอร์ชันอัปเกรด! คู่มือนี้จะช่วยให้คุณเข้าใจฟีเจอร์ต่างๆ และวิธีการใช้งานทั้งในฐานะ **ผู้ใช้งานทั่วไป** และ **ผู้ดูแลระบบ (Admin)** ครับ

---

## 1. สำหรับผู้ใช้งานทั่วไป (User Guide)

### 🔍 การค้นหาและจองห้องประชุม
1.  **เลือกวันและเวลา**: ไปที่หน้า "ค้นหาห้อง" เพื่อเลือกช่วงวันที่คุณต้องการใช้งาน
2.  **ตรวจสอบสถานะห้อง**: ระบบจะแสดงผลเฉพาะห้องที่ "ว่าง" ในช่วงเวลานั้นๆ หากห้องถูกจองไปแล้ว ระบบจะขึ้นเตือนว่า "Occupied"
3.  **กรอกรายละเอียดการจอง**:
    *   **เลือกวันจัดเตรียม (Setup Date)** และ **วันใช้งานจริง (Usage Date)** ผ่านปฏิทินที่จัดระเบียบใหม่ให้กระชับ
    *   **เลือกอุปกรณ์ (Amenities)**: ติ๊กเลือกอุปกรณ์ที่ต้องการใช้ (เช่น โปรเจคเตอร์, ไมค์)
    *   ระบุวัตถุประสงค์และจำนวนผู้เข้าร่วม
4.  **ยืนยันการจอง**: กดปุ่ม "ยืนยันคำขอ" ระบบจะส่งเรื่องไปหาผู้ดูแลระบบทันที

### 📜 การจัดการรายการจองของตนเอง
*   ไปที่หน้า **"การจองของฉัน" (My Bookings)** เพื่อดูสถานะคำขอ (รออนุมัติ / อนุมัติแล้ว / ปฏิเสธ)
*   **ยกเลิกการจอง**: หากต้องการยกเลิกคำขอที่ยัง "รออนุมัติ" สามารถกดปุ่มยกเลิกได้ทันที เพื่อคืนสิทธิ์ให้คนอื่นจองต่อ

### 🔔 ระบบแจ้งเตือน
*   คุณจะได้รับแจ้งเตือนที่รูประฆังมุมขวาบน เมื่อสถานะการจองของคุณมีการเปลี่ยนแปลง (เช่น ได้รับการอนุมัติ หรือถูกปฏิเสธพร้อมระบุเหตุผล)

---

## 2. สำหรับผู้ดูแลระบบ (Admin Guide)

### 📋 การพิจารณาอนุมัติ (Admin Dashboard)
*   รายชื่อการจองที่รอการตรวจสอบจะปรากฏในหน้า **"รายการรออนุมัติ"**
*   **อนุมัติ (Approve)**: กดอนุมัติเพื่อผ่านขั้นตอนไปขั้นถัดไป หรืออนุมัติถาวร
*   **ปฏิเสธ (Reject)**: หากต้องปฏิเสธ ระบบจะบังคับให้กรอก "เหตุผลการปฏิเสธ" เพื่อแจ้งให้ผู้จองทราบผ่านระบบแจ้งเตือน

### 🏨 การจัดการห้องประชุม (Room Management)
*   **เพิ่มห้องใหม่**: สามารถอัปโหลดรูปภาพได้หลายรูปพร้อมกัน
*   **จัดการอุปกรณ์**: มีปุ่ม **"Quick Add"** เพื่อเพิ่มอุปกรณ์ยอดฮิตอย่างรวดเร็ว
*   **แก้ไขข้อมูล**: สามารถเปลี่ยนชื่อห้อง, ความจุ, หรือปิดห้องเพื่อซ่อมบำรุง (Maintenance) ได้

### 📊 รายงานและสถิติ (Reports & Analytics)
*   ดูภาพรวมการใช้งานห้องประชุม เช่น ห้องไหนถูกจองบ่อยที่สุด และสัดส่วนสถานะการจองทั้งหมดในระบบ

### 👥 การจัดการสิทธิ์ผู้ใช้ (User Management)
*   แอดมินสามารถแต่งตั้งหรือถอดถอนสิทธิ์ผู้ดูแลระบบคนอื่นๆ ได้จากหน้านี้โดยตรง

---

## 3. ฟีเจอร์พิเศษ (Smart Features)

*   **Real-time Update**: ระบบใช้ Supabase Channels ข้อมูลบนหน้าจอจะอัพเดททันทีเมื่อมีการเปลี่ยนแปลงสถานะโดยไม่ต้องกด Refresh
*   **Conflict Validation**: ระบบจะไม่อนุญาตให้จองวัน/เวลาที่ทับซ้อนกับคนอื่น (เช็คตั้งแต่ในหน้าจอง)
*   **Dark Mode Support**: รองรับการใช้งานทั้งโหมดสว่างและโหมดมืดอย่างสมบูรณ์

---

## 🛠️ ข้อมูลทางเทคนิคสำหรับนักพัฒนา
*   **Frontend**: React (Vite) + Tailwind CSS
*   **Backend/Database**: Supabase (PostgreSQL + Real-time)
*   **Authentication**: Supabase Auth
*   **Storage**: Supabase Storage (สำหรับภาพห้องประชุม)
